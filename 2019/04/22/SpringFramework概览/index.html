SpringFramework概览

  • 2019年04月22日
  • duyu

概览

SpringFramework3的框架结构图 (重要的里程碑版本) (SpringFramework4去掉了Struts, 增加了WebSocket, Messaging) (SpringFramework5增加了WebFlux, 最低java8, 所以支持响应式编程)

参考: Spring Framework5中的新特性 Spring Framework架构

SpringFramework一共分为五部分

  1. Core springframework核心
  2. AOP 面向切面部分
  3. DataAccess 数据访问整合部分
  4. Web webmvc部分
  5. Test 测试部分

最核心(IOC)就Core模块中的三个 Core, Beans, Context 它通过DI注入方式, 解决了Bean的生命周期统一管理 图中每个块即为一个jar包, instrumentation有两个包, spring-context-support未在图中列出, 一共19个jar包 4.0版本取消了Struts, 增加了WebSocket和Messaging, 所以有20个jar包

下面我们就以各个包的作用以及依赖关系做出解释

Core核心模块 (IOC依赖注入)

- Core核心模块 - 特点
spring-core 依赖注入IoC与DI的最基本实现, 只依赖common-logging
spring-beans Bean工厂与bean的装配, 只依赖性spring-core
spring-expression spring表达式语言, 只依赖性spring-core
spring-context spring的context上下文即IoC容器, 它依赖了4个模块spring-core,spring-beans,spring-expression,spring-aop

所以, 依赖了spring-context就相当于可以使用Spring的IOC和AOP

AOP面向切面模块 (动态代理)

- AOP模块 - 特点
spring-aop 面向切面编程 (动态代理), 依赖spring-core, spring-beans, aopalliance
spring-aspects 集成AspectJ (编译期静态代理)
spring-instrument 提供一些类级的工具支持和ClassLoader级的实现, 用于服务器; instrument是jdk5引入,jdk6发扬光大的, ASM,CGLIB,javassist,jdk动态代理之流只能通过反射或字节码方式创建一个新类, 但javaAgent是 JVM虚拟机级别的动态AOP代理, 在载入jvm前动态修改class的字节码, 注意是在原始字节码基础上做修改, 不会生成一个新的字节码文件, 也没有必须有接口等限制 参考: Java Instrument原理
spring-instrument-tomcat 针对tomcat的instrument实现

需要注意aspectj的支持需要单独依赖spring-aspects

DataAccess数据访问模块

- 数据访问模块 - 特点
spring-jdbc jdbc的支持
spring-tx 事务控制, 定义了七种事务传播行为
spring-orm 对象关系映射，集成orm框架; 主要针对JDO ,ibatis, hibernate…
spring-oxm 对象xml映射, 用以支持xml方式
spring-jms java消息服务

Web模块

- 数据访问模块 - 特点
spring-web 基础web功能, 如文件上传, servlet等
spring-webmvc mvc实现, 通过策略模式, 所有都变成可配置, 视图技术支持包括JSP、Velocity、Tiles、iText、POI、FreeMarker
spring-webmvc-portlet 基于portlet的mvc实现, 类似Fragment片段组成的单页后端web应用, 与安卓Fragment, 前端单页SPA相对应
~~spring-struts~~ 与struts的集成, 不推荐, spring4已移除
spring-websocket springframework4开始支持

附注 : protlet类似fragment片段, 可以理解为服务端的单页面web应用, 和servlet传统模式有本质区别 WebSphere容器支持了 JSR 168 Portlet API 的 Portlet 容器

Test测试模块

- 测试模块 - 特点
spring-test spring测试, 提供junit与mock测试功能
spring-context-support spring额外支持包, 比如邮件服务、视图解析等

springframework4.0

去掉spring-struts的支持, 新增spring-websocket, spring-messaging两个模块 (还支持groovy, spring-beans-groovy)

- 4.0新增模块 - 特点
spring-websocket 为web应用提供的高效通信工具
spring-messaging 用于构建基于消息的应用程序

总结

根据以上的模块的依赖关系

  1. 若只想使用sping的IOC和AOP, 则只需要引用spring-context
  1. 若想使用spingmvc, 则只需要引用spring-webmvc
