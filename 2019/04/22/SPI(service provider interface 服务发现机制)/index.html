SPI(service Provider Interface 服务发现机制)

  • 2019年04月22日
  • duyu

SPI全称为(Service Provider Interface) ，是JDK内置的一种服务提供发现机制；主要被框架的开发人员使用，比如java.sql.Driver接口，数据库厂商实现此接口即可，当然要想让系统知道具体实现类的存在，还需要使用固定的存放规则，需要在classpath下的META-INF/services/目录里创建一个以服务接口命名的文件，这个文件里的内容就是这个接口的具体的实现类；日志门面框架和DriverManager数据库驱动都是基于此。

spi是一种动态发现并替换的机制, 比如运行时动态添加接口实现,主要针对厂商或者插件; 典型的比如数据库驱动Driver, slf4j日志框架(外观模式) 基于接口编程 + 策略模式 + 配置文件

1.定义一组接口 com.doyo.learn.spi.IHello

2.接口的多个实现 com.doyo.learn.spi.impl.HelloV1Impl com.doyo.learn.spi.impl.HelloV2Impl

3.在 src/main/resources/ 下建立 /META-INF/services 目录,新增一个以接口命名的文件 /META-INF/services/com.doyo.learn.spi.IHello 文件的内容是实现类的全名,可以写上多个实现, 比如: com.doyo.learn.spi.impl.HelloV1Impl com.doyo.learn.spi.impl.HelloV2Impl

4.使用ServiceLoader来加载指定配置文件中的实现

例如有一个开源搜索模块基于接口编程规范,可能基于文件系统,可能基于数据库; 接口如下:

A公司提供文件搜索实现com.A.spi.impl.FileSearch A公司发布jar时,文件META-INF/services/org.xyz.spi.ISearch中填写如下内容:com.A.spi.impl.FileSearch

B公司提供数据库搜索实现com.B.spi.impl.DatabaseSearch B公司发布jar时,文件META-INF/services/org.xyz.spi.ISearch中填写如下内容:com.B.spi.impl.DatabaseSearch

最后使用者会通过开源搜索jar包(门面模式)的工厂类来使用

总结: 这是一种接口与实现分离解耦,提升可扩展性,轻松实现功能模块的可插拔;开发者无须手动注入实现,只需引入具体实现jar包 + spi接口jar包 参考: java中的SPI机制 jdbc的设计 spi服务加载的手工实现
